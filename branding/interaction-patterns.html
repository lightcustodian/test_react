.card-interactive {
  transition: all var(--transition-base);
  cursor: pointer;
}

.card-interactive:hover {
  transform: translateY(-8px) scale(1.02);
  box-shadow: var(--shadow-hover);
}

Strategic Color Transitions

Business Logic: Icon colors transition between Blue (trust/expertise) and Green (growth/innovation) to reinforce messaging

Icon Transition Patterns

Trust Icons (Green → Blue)
→ Hover to see Blue transition
Expertise Icons (Blue → Green)
→ Hover to see Green transition
/* Icon color transitions triggered by CARD hover */
.card-interactive:hover .icon-trust {
  background: var(--color-bg-blue); /* Green → Blue */
}

.card-interactive:hover .icon-expertise {
  background: var(--color-bg-green); /* Blue → Green */
}

Button Interactions

Consistent Feedback: All buttons provide clear hover feedback with subtle elevation and color shifts

Button States

.btn-primary:hover {
  background: var(--color-dark-green);
  transform: translateY(-2px);
  box-shadow: var(--shadow-md);
}

Implementation Guidelines

DO: Use .card-interactive for 90%+ of interactive cards | Apply consistent timing with var(--transition-base) | Follow strategic color mapping
DON'T: Mix different hover patterns | Use hardcoded transition values | Apply transitions to non-interactive elements

Transition Timing

:root {
  --transition-fast: 150ms ease;
  --transition-base: 300ms ease;
  --transition-slow: 500ms ease;
}

/* Use base timing for most interactions */
.card-interactive {
  transition: all var(--transition-base);
